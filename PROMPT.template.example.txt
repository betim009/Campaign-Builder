Atalhos operacionais (recomendado ao iniciar uma sessão):

1) Validar stack local

- `docker compose ps`
- `curl http://localhost:3001/healthz`

2) Validar Meta (sem side effects)

- `curl http://localhost:3001/api/meta/status`
- `curl -X POST http://localhost:3001/api/meta/validate -H 'Content-Type: application/json' -d '{}'`
- Diagnóstico de Pages: `curl "http://localhost:3001/api/meta/pages?metaAdAccountId=act_<id>"`

3) Blocker comum (P4/P5)

- Creative REAL/Ad REAL exige `META_PAGE_ID` (env) **ou** `pageId` no UI do `/meta-test`.
- `docker-compose.yml` faz passthrough de `META_PAGE_ID` e `META_INSTAGRAM_ACTOR_ID`; após ajustar `.env`, use `docker compose restart backend`.
- `/api/meta/pages` tenta múltiplas fontes (`me/accounts`, `act_*/promote_pages`, `me/businesses` + `owned_pages`). Se tudo vier vazio, o token provavelmente não tem acesso a uma Page.

Regras de segurança (sempre):

- Token Meta nunca vai ao frontend.
- Toda criação REAL deve permanecer `PAUSED` (forçado no backend).
- Não remover fallback `STUB` nem troubleshooting operacional.
